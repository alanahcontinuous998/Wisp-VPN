# Как помочь проекту

Спасибо за интерес к Wisp VPN! Проект открытый и держится на энтузиастах — любой вклад ценен.

## Способы помочь

- 🐛 **Сообщить о баге** — через [Issues](https://github.com/PostmanReminisce/wisp-vpn/issues/new/choose), шаблон «Bug report»
- 💡 **Предложить фичу** — через [Issues](https://github.com/PostmanReminisce/wisp-vpn/issues/new/choose), шаблон «Feature request»
- 🌍 **Перевод/локализация** — сообщите в issue, какой язык хотите добавить
- 💻 **Код** — см. ниже
- 📖 **Документация** — правки README, docs/*, опечатки — всегда welcome

## Процесс работы с кодом

1. Сделайте fork репозитория
2. Создайте ветку от `main`: `git checkout -b feature/короткое-название`
3. Вносите изменения небольшими логичными коммитами
4. Убедитесь, что не закоммитили реальные ключи/секреты (см. `.gitignore`)
5. Откройте Pull Request, заполнив шаблон PR — опишите **что** и **зачем**

## Стиль коммитов

Желательно придерживаться формата `тип: краткое описание`, например:

```
fix: корректно закрывать туннель при выходе из приложения
docs: добавить раздел про kill switch в FAQ
feat: автовыбор сервера по пингу
```

## Обсуждение

Для вопросов и предложений, не требующих оформления issue, — вкладка [Discussions](https://github.com/PostmanReminisce/wisp-vpn/discussions) (если включена) или issue с меткой `question`.

Правила поведения в проекте описаны в [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).
